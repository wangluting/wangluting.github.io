Ctci 读书笔记（一）

发表于 2016-05-14   |   分类于 面试 ， 读书笔记   |  

好记性不如烂笔头。
今天开始读Cracking the coding interview。
希望能在8月份去美国之前把这本书读完一遍。

面试前

  • 练习在白板上手写代码，可以通过在纸上写代码，然后再在编译器上写来练习
  • 对自己的简历上的项目很熟悉，对于每一个项目，想出来一个最难的和一个最有趣的细节，可以和面试官讨论的
  • 不要尝试去背答案，而是通过编程练习来强化你的技能
  • 说话大声，与面试官积极交流，让面试官了解你的想法，他们才能给你一些指导
  • 最后，要记住，Interviews are hard! 尽管你在面试过程中可能被问了比较简单的问题，但是简单的问题并不意味着能拿到offer。拿到offer并不是因为你在面试中回答对了所有的问题，而是answering questions better than others. 因此，当你拿到了一个比较tricky的问题时不要慌张，因为别人拿到这道题也是很难的，你要做的就是表现的比其他人出色，就够了。

面试背后的那些事：

面试过程就像是一个黑箱子，面试者走进去，接收了一些问题，然后走出来，从箱子里拿出一个offer or not。面试者想知道箱子里究竟发生了什么?

  1. 面试官是如何做决定的
  2. 面试官之间会讨论吗
  3. 公司真正想招什么样的人才

微软:

  • 4-5轮on-site, 一般是两个不同的团队。当完成了一个团队的所有面试后，如果进行到了hr面试，可以认为前面的面试都过了，决定权在最后这个人手里
  • 微软想要的人是对技术有激情的人,Geeks
  • “Why do you want to work for Microsoft?”
    Microsoft want to see that you are passinate about technology. For example, “I’ve been using Microsoft software as long as I can remember, and I am really impressed at how Microsoft manages to create a product that is universally excellent”

亚马逊：

  • 1-2轮电话面试，写一段简单的代码并在电话中大声朗读出来。然后一些比较广泛的问题，去挖掘面试者熟悉哪一方面；通过了的话，就会邀请去总部on-site。一般是根据你的简历和电话面试结果，有1-2个组会对你进行面试。在白板上写代码。
  • Amazon is a web-based company, that means they care about scale. Make sure you prepare for questions in “large” scale.
  • Amazon tends to ask a lot of questions about OOD.

谷歌：

  • 首先是电面，会问一些难题。On-site会有4-6个人对你进行面试，其中一个是lunch interviewer. 他并不会给出feedback，所以可以问他“诚实”的问题。其他面试官的反馈会被汇总到一起，根据面试者的Analytical Ability, Coding, Experience, Communication四个方面的得分综合判定，不太考虑电面的成绩。
  • Google is web-based company, cares about how to design a scalable system from “System Design and Memory Limits”
  • bit manipulation

苹果

  • 首先是几轮电面。如果被邀请on-site，一般是6-8个面试，1-on-1 or 2-on-1. 白板写代码。
  • 需要了解苹果的文化，产品，如果知道具体面试的团队，可以准备一些对他们组产品的建议，表现出自己对苹果公司的热情。

简历建议：

面试官期待从简历中看到的两点:
1 你是否够聪明
2 你是否会写代码
