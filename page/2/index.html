Chapter 7 Object Oriented Design

面向对象设计在面试中经常会遇到，对这一类问题的解答可以反映出程序员写代码的功力。这类问题常常被故意描述地很模糊，面试官关注于面试者能否做出假设，能否向面试官确认问题条件。因此面试者遇到OOD的问题时要向面试官确认题目，排除歧义。
软件中的面向对象和真实生活中的面向对象很相像。需要弄清楚以下几个问题：

目标是什么？如果是设计一副牌，那么你需要向面试官确认，要拿这一副牌做什么？是否可以假设这副牌拿来做一些通用的纸牌游戏？如果是要设计一个停车场，那么这个停车场是否是要得到某个停车位是否被占用，同一类型的车在停车场中有多少辆之类的
然后，想一想需要哪些父类，又需要哪些子类？例如设计一个停车场，需要设计车辆，停车场，停车位，停车计时器等父类。其中车辆有多个子类，停车位是有残疾人停车位等子类。
接着，想一想自己是否遗漏了一些信息。例如如何设计停车限制，分别依赖于时间或者停车费。进一步丰富类的种类和内容。
最后，具体化这些类。设计方法获得停车时间，停车位是否被占用等。思考如何表示数据可以使这些方法更加高效。

题目

7.1 Design the data structures for a generic deck of cards Explain how you would sub- class it to implement particular card games.

Chapter 6 Brain Teasers

在面试中可能会被问到一些脑筋急转弯的问题（尽管一些公司规定在面试过程中禁止问这类的题目）。遇到这类问题的时候不要慌，因为有些时候面试官只是想看你对待这类智力题的反应，他们并不要求面试者能立马想出完美的答案来。这时候要多跟面试官交流，让他们直到你是怎么思考的，而不是想不出答案也不说话。

实际上，这类的问题通常都与计算机知识相关（特别注意题目出现的数字，比如说1000，就要想到1024，2的10次方之类的）。好的建议是将问题简化，看看能不能解决一个子问题或者想出在某个特例下的解法，然后再尝试将问题扩大到更普遍的解法。

举个例子，你现在需要准确的15分钟来煎鸡蛋，但是你没有钟表。你唯一有的是两条绳子。这两条绳子有个特点，那就是将任意一根绳子点燃直到燃尽所花的时间是准确的一个小时（60分钟）。绳子是不均匀的，即将绳子剪成两半点燃其中的一半所花的时间不是准确的30分钟。请问你该怎么计时来煎鸡蛋？

这道题看起来不可解，但是通过将问题简化就可以想出解法。

现在可以精确的计算60分钟（通过燃烧一条绳子）
如何精确计算30分钟呢？ 即如何让绳子的燃烧速度变为之前的两倍？
对，从一条绳子的两端同时点燃
那怎么精确计算15分钟呢？
这样想，如果我们有一条燃尽需要30分钟的绳子，那么通过两端同时点燃就可以精确计算15分钟了
那如何可以得到一条燃尽需要30分钟的绳子呢？
对，让两个绳子同时点燃，不同的是，一条绳子从一头燃烧，另一条绳子从两段同时燃烧。当一条绳子
燃尽的时候，我们就直到已经过去了30分钟了，即另一条绳子燃尽还需30分钟，此时再把另一条绳子的
另一头点燃，即剩下的绳子在15分钟会燃尽，此时开始煎鸡蛋就可以了。

Chapter 5 Bit Manipulation

位运算就是直接对整数在内存中的二进制位进行操作。由于程序中所有的数在计算机中都是以二进制的形式存储的，因此位运算特别快，而且代码比较简洁。
位运算有6种：&（与），｜（或），～（反），^（异或），<< >> （左右移）

Chapter 4 Trees and Graphs

树和图是面试中经常被问到的两种数据结构，常以一下两种方式出现：

  • Implement a tree/find a node/delete a node/other well known algorithm
  • Implement a modification of a known algorithm
    在面试前，务必理解掌握树的相关算法。

实现

Tree

需要注意的是，对于binary tree,面试者经常会把它默认为binary search tree, 然而面试官经常就只是指的是binary tree. 因此面试者要仔细听是否有关键词”search”.实在不清楚的话，需要向面试官提问。
关于二叉树，必须熟练掌握的算法：

  • 先序遍历(Pre-order)
  • 中序遍历(In-order)
  • 后序遍历(Post-order)
  • 插入节点(向二叉搜索树中插入节点，和根节点比较，如果大于根节点就向右遍历，小于根节点就向左遍历，直到找到一个空位插入。虽然二叉搜索树中节点的调整和删除很少在面试中出现，但是如果你知道具体是如何做的，会让你的表现较其他面试者更加突出)

转眼，已近快五月底了。距离自己3月初投出的第一封简历开始，已经接近3个月了。回首过往，这90天日日夜夜，其中的辛酸与喜悦，也许只有自己才能体会到。我并不是什么计算机大牛，开始转行计算机也才只有半年的时间，并没有可以拿的出手的相关项目，以至于刚开始的时候看着自己的简历，连过简历关的自信都没有。最终能拿到Nvidia的offer，并进入了百度，阿里和网易游戏的终面，靠的算是自己的努力和幸运吧。
