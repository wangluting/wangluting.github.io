Ctci 读书笔记（一）

发表于 2016-05-14   |   分类于 面试 ， 读书笔记   |  

好记性不如烂笔头。
今天开始读Cracking the coding interview。
希望能在8月份去美国之前把这本书读完一遍。

面试前

  • 练习在白板上手写代码，可以通过在纸上写代码，然后再在编译器上写来练习
  • 对自己的简历上的项目很熟悉，对于每一个项目，想出来一个最难的和一个最有趣的细节，可以和面试官讨论的
  • 不要尝试去背答案，而是通过编程练习来强化你的技能
  • 说话大声，与面试官积极交流，让面试官了解你的想法，他们才能给你一些指导
  • 最后，要记住，Interviews are hard! 尽管你在面试过程中可能被问了比较简单的问题，但是简单的问题并不意味着能拿到offer。拿到offer并不是因为你在面试中回答对了所有的问题，而是answering questions better than others. 因此，当你拿到了一个比较tricky的问题时不要慌张，因为别人拿到这道题也是很难的，你要做的就是表现的比其他人出色，就够了。

面试背后的那些事：

面试过程就像是一个黑箱子，面试者走进去，接收了一些问题，然后走出来，从箱子里拿出一个offer or not。面试者想知道箱子里究竟发生了什么?

  1. 面试官是如何做决定的
  2. 面试官之间会讨论吗
  3. 公司真正想招什么样的人才

微软:

  • 4-5轮on-site, 一般是两个不同的团队。当完成了一个团队的所有面试后，如果进行到了hr面试，可以认为前面的面试都过了，决定权在最后这个人手里
  • 微软想要的人是对技术有激情的人,Geeks
  • “Why do you want to work for Microsoft?”
    Microsoft want to see that you are passinate about technology. For example, “I’ve been using Microsoft software as long as I can remember, and I am really impressed at how Microsoft manages to create a product that is universally excellent”

亚马逊：

  • 1-2轮电话面试，写一段简单的代码并在电话中大声朗读出来。然后一些比较广泛的问题，去挖掘面试者熟悉哪一方面；通过了的话，就会邀请去总部on-site。一般是根据你的简历和电话面试结果，有1-2个组会对你进行面试。在白板上写代码。
  • Amazon is a web-based company, that means they care about scale. Make sure you prepare for questions in “large” scale.
  • Amazon tends to ask a lot of questions about OOD.

谷歌：

  • 首先是电面，会问一些难题。On-site会有4-6个人对你进行面试，其中一个是lunch interviewer. 他并不会给出feedback，所以可以问他“诚实”的问题。其他面试官的反馈会被汇总到一起，根据面试者的Analytical Ability, Coding, Experience, Communication四个方面的得分综合判定，不太考虑电面的成绩。
  • Google is web-based company, cares about how to design a scalable system from “System Design and Memory Limits”
  • bit manipulation

苹果

  • 首先是几轮电面。如果被邀请on-site，一般是6-8个面试，1-on-1 or 2-on-1. 白板写代码。
  • 需要了解苹果的文化，产品，如果知道具体面试的团队，可以准备一些对他们组产品的建议，表现出自己对苹果公司的热情。

简历建议：

面试官期待从简历中看到的两点:
1 你是否够聪明
2 你是否会写代码

一些无关的信息，比如说个人爱好不要出现在简历上。简历上只保留最打动人的，最相关的项目经验。不相关的信息会毁了你的简历并且分散面试官的耐心。 试着这样写：

  • Accomplished X by implementing Y which led to Z
  • show what you did, how you did it, what the result were. Try to make the results “measurable” somehow.
  • 在简历中列出2-4个最重要的projects. State what the project was, which language or technologies it employed, and whether it was an individual project or a team project. 非课程的项目会更加preferable.
  • 简历上如果要列出会用的软件或工具，不要写MS之类的技能，可以列出VS, Eclipse, Linux
  • 对于编程语言，列出常用的几种语言并且标注上能力级别。例如：Java(expert), C++(proficient), JaveScript(prior experience)
  • 对于Non-Native applicant, 不要在简历中出现typo

技术之外的准备：

面试中，面试官为了了解面试者的性格，或是缓和紧张气氛，会问一些behavioral questions.
例如：tell me about a time when…
建议面试者在面试前列出这样一张表:

有了这张表，在面试中被问起简历里相关的项目后，就会come up with an appropriate story effortlessly.

  • 当被问起weakness时，请给出一个真正的缺点，并且说明你是如何克服的
  • 你并不是在简单的回答面试官的每一个问题，而是在telling yourself.仔细想一想每一个回答如何展示出真正的自己

适合问面试官的问题：

在面试结束后，一般都会留给面试者向面试官提问的机会，提问的质量有可能会成为影响他们是否给你发offer的因素。问题一般可以分为3类：
1.Genuine questions：
这一般是面试者真正想知道的

  1. How much of your day do you spend coding?
  2. How many meetings do you have every week?

2.Insightful questions:
这类问题一般能表明面试者deep knowledge of programming or technologies

  1. I noticed that you are using technology X. How do you handle problem Y?
  2. Why did the product choose to use the X protocol over the Y protocol? I know it has benefits like A, B, C, but many companies choose not to use it because of issue D.

3.Passion questions:
这类问题能表现面试者passion for technology

  1. I am very interested in scalability. Did you come in with a background in this, or what opportunities are there to learn about it?
  2. I’m not very famaliar with technology X, but it sounds like a very interesting solution. Could you tell me a bit more about how it works.

技术方面的准备：

该如何准备技术问题，以下是几点建议：

  • 自己解决问题， I mean, really try to solve it. 有一些问题被设计出来就是很难的，that’s Ok. 在解决问题时仔细想想时间和空间复杂度，问问自己能否牺牲空间来换取时间，或者反之
  • 在纸上写代码，前面已经提到过了
  • 将纸上的代码敲进电脑，记下来手写代码中那些错误的地方，keep in mind and make sure it won’t happen in interview
  • 模拟面试

一般来说，特别复杂的算法都不会出现在面试中，因为面试官也不记得这些算法的细节。
对于面试者必须掌握的知识：

Data Structure Algorithm Concepts
Linked List Breadth First Search Bit manipulation
Binary Tree Depth First Search Sinngleton Design Pattern
Tries Binary Search Factory Design Pattern
Stacks Merge Sort Memory(Stack vs Heap)
Queues Quick Sort Recursion
Vectors/ArrayLists Tree Insert/Find/etc Big O problem
Hash Tables

对于上面表格列出的这些要点，需要知道如何实现它们和使用，并且知道时间和空间复杂度。

前面都是面试之前的准备，记下来讲讲面试中如何表现

面试中

刚开始的时候，面试官通常会问一些behavioral questions. 对于这类问题的回答，有以下几点建议：

  • be specific, not arrogant. Specificity 意味着客观陈述事实并由面试官来判断.
  • limit details. 试着只陈述key points. 最后向面试官表明：I can go into more details if you’d like.
  • ask good questions，前面已经总结了应该问面试官哪些问题
  • structure answers using S.A.R. Situation, Action, Response.回答问题的时候，首先表明situation，然后解释采取了什么action,最后描述results。

当进入到真正的面试，面试官开始问technical questions

Five steps to a technical questions:

  1. 就题目不清晰的地方向面试官提问
  2. 设计算法
  3. 写伪码，并且告诉面试官你先写伪码，否则面试官可能认为你不会写代码
  4. 真正写代码，不要写太快，也不要写太慢
  5. 检查代码是否有出错的地方

向面试官提问：

What are the data types?
How much date is there?
What assumption do you need to solve the problem
Who is the user?

设计算法：
考虑以下5点：

  1. space complexity and time complexity
  2. what happens if there is much data
  3. Does the design cause other issue?
  4. If there is other issue, did you make the right trade-off
  5. If they gave you specific data, have you leveraged that information?

面试后

拿到了offer后，根据自己的职业规划，以及比较各个公司的package选择心仪的offer

十个面试者容易犯的错误

  • 在电脑上练习写代码
  • 不准备技术外的问题
  • 不做模拟面试
  • 尝试背答案
  • 说的太多
  • 说的太少
  • 写代码太急
  • 不检查代码
  • 代码风格不佳
  • 容易放弃
