Chapter 13/14 C++ & JAVA

一般来说，面试官不会关心你用什么语言写代码。但是如果你在简历里面说了自己会某种语言, 则面试官可能会问你相关的问题。不过不用担心，你并不需要很清楚语言的细节，因为面试官也不一定很懂。但是你需要知道基本知识，例如c++中的指针和Java中的类等。

C++ Java
面向过程/对象 完全面向对象
编译型语言,生成的目标文件针对特定的cpu 解释型语言, 类文件平台无关
支持多重继承 单一继承
可以使用指针访问内存 无法通过指针访问内存
程序员管理堆内存 自动内存管理, 垃圾回收机制
支持操作符重载 不支持操作符重载
支持无符号数 没有无符号数

C++主要用于系统性应用程序的设计, 大型游戏和桌面应用；
Java语法更清晰，规模更小，跨平台，适用于服务端和客户端应用程序开发。

如果在面试中对某种语言的某种实现不太了解，试着思考自己熟悉的语言是怎么实现的；如果要自己来实现会如何做等。如果实在想不出来，试着向面试官解释自己暂时不记得了，但是自己可以通过一个例子来展示它的用法，例如这段代码……

Chapter 11 Testing

测试相关的问题不仅仅只会出现在测试工程师的面试中，而且会出现在一般的程序员面试中，用来考察程序员对代码测试的能力。一个好的程序员要知道如何测试自己写的代码。
常见的测试问题包括：

  1. 解释如何测试真实世界中的物体（笔，夹子等）
  2. 解释如何测试软件（浏览器）
  3. 编写测试代码

针对第一种情况，我们可以举出一个例子：问如何测试夹子（paperclip）?
遇到这种问题，首先应该弄清楚：这个夹子的用途是什么，以及使用的人群是哪类？ 这个夹子可能是给教师拿来夹试卷的，也可能是给手工艺者拿来设计折纸的。不同的用途会有不同的答案。在确保弄清楚用途和使用者后，接着要考虑的是：

  1. 测试用例的范围是什么？例如：夹两张纸，最多夹30张纸。
  2. 如何定义不合格？
  3. 除了要求的用途外，是否还有其他用途。
  4. 测试的极端条件是什么什么？高温，低温等

Chapter 9 Sorting and Searching

排序和查找问题是面试中的高频问题。当我们遇到一个排序问题时，好的做法是比较不同的排序算法然后找到一个最适合于该应用背景的。例如：现在有很大一批人，让你根据他们的年龄排序。从这个问题我们应该提炼出2个关键点：

  1. 待排序的对象有很多
  2. 排序的值是人的年龄，而年龄这个数字的范围是比较小且有界的
    从以上两点我们可以得出最适合的排序应该是桶排序，时间复杂度为O(n), 空间复杂度也是常数级。

Chapter 8 Recursion

面试中会遇到许多可以用递归解决的问题，这类问题都有一些共同点：那就是它们都可以拆解成小的问题，即解决这个问题可以通过解决它的子问题来解决。可以用递归解决的问题通常（但不总是）有规律可循，例如：设计一个算法去解决第n个；写代码列出前n个；实现一个方法去计算所有的。

遇到这类问题，通常可以通过以下的步骤去分析：

  1. 想一想子问题是什么？有多少个子问题
  2. 解决base case. 如果需要f(n),那么base case就是f(0)或f(1). 它的值一般是hard-code，可以从题目推出
  3. 由base case继续解决f(2),f(3)
  4. 仔细思考f(3)是如何从f(2)得到的，即弄明白由子问题如何推得大问题的解
  5. 得到f(n)这个通用的解
    自底向上的解法通常十分直接。有的时候自顶向下的方法也可行。

在解决递归问题的时候，有两点需要注意的：

  1. 所有用递归解决的问题都可以用迭代来解决（虽然用迭代的解法可能会使得代码更加复杂）。在用递归解决问题之前，问问自己如果用迭代来解决会有多复杂。可以和你的面试官讨论trade-off.
  2. 递归的解法比较浪费空间，你可以想想每一次调用都往栈里面加了一层，意味着如果你的代码时间复杂度为o(n)，那么空间复杂度也至少为o(n).这也是为什么迭代的解法更好。
