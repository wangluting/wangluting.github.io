Chapter 16 Low level

这一大类的问题主要考察的是面试者对计算机相关的一些基本概念的理解。对于需要面试者对于这类相关问题的理解程度则取决于公司的需要。一般来说，需要了解的只是概念层，能够解释清楚相关的概念就好了。

端序

端序是字节的存储顺序。如果数据是单字节的，那么如何存储就无所谓了，然后对于多字节数据，比如说int，double等，就要考虑存储的顺序了。
字节序是硬件层面的东西，对于软件是透明的，只和处理器架构有关，和编程语言无关；x86是小端序
大端序：数据的高位字节存放在地址的低端，低位字节存放在地址的高端
小端序：数据的高位字节存放在地址的高端，低位字节存放在地址的低端
好的记忆方法是：大端序按照数字的书写顺序进行存储的

内存

  1. 代码区：存放函数体（类成员函数和全局函数）的二进制代码。
  2. 栈区（stack）：由编译器自动分配释放 ，存放为运行函数而分配的局部变量、函数参数、返回数据、返回地址等。其操作方式类似于数据结构中的栈。
  3. 堆区（heap）：一般由程序员分配释放（new/delete malloc/free）， 若程序员不释放，程序结束时可能由 OS 回收 。分配方式类似于链表。
  4. 全局区（静态区）：存放全局变量、静态数据、常量。程序结束后由系统释放
  5. 常量区： 常量字符串就是放在这里的。 程序结束后由系统释放。

malloc/free与new/delete的区别

new是运算符，malloc是库函数
运算符使用是否正确，编译器在编译扫描分析时就可以判定
库函数是已编译的代码，编译器不会编译检查，由链接器将库同用户写的代码合成exe文件

对于非内部数据类型的对象而言，光用maloc/free无法满足动态对象的要求。对象在创建的同时要自动执行构造函数，对象在消亡之前要自动执行析构函数。由于malloc/free是库函数而不是运算符，不在编译器控制权限之内，不能够把执行构造函数和析构函数的任务强加于malloc/free。因此C++语言需要一个能完成动态内存分配和初始化工作的运算符new，以及一个能完成清理与释放内存工作的运算符delete。由于malloc/free是库函数，不是运算符，他们不能执行构造函数和析构函数，只是负责分配内存。

new 建立的是一个对象，malloc分配的是一块内存。 new建立的对象你可以把它当成一个普通的对象，用成员函数访问，不要直接访问它的地址空间。 malloc分配的是一块内存区域，就用指针访问好了，而且还可以在里面移动指针。

Chapter 15 Databases

涉及到数据库相关的问题有：写sql语句；针对某一场景设计数据库等。
举个例子：假设现在需要你去为一个国际的房屋租赁公司设计数据库。
需要有的表有：property, building, apartment, manager, tenant.
需要弄清楚它们之间的关系：one property could have multiple manager, a manager could hold many properties; a building could only be one property, a apartment could only be one building; a apartment could have multiple tenants, a tennat could rent many apartment.
有的时候，表很大在做表的连接时就会比较慢，这时候需要考虑将一些信息放在同一个表中，因此有些数据可能会重复存储。这是一个trade-off.

Chapter 13/14 C++ & JAVA

一般来说，面试官不会关心你用什么语言写代码。但是如果你在简历里面说了自己会某种语言, 则面试官可能会问你相关的问题。不过不用担心，你并不需要很清楚语言的细节，因为面试官也不一定很懂。但是你需要知道基本知识，例如c++中的指针和Java中的类等。

C++ Java
面向过程/对象 完全面向对象
编译型语言,生成的目标文件针对特定的cpu 解释型语言, 类文件平台无关
支持多重继承 单一继承
可以使用指针访问内存 无法通过指针访问内存
程序员管理堆内存 自动内存管理, 垃圾回收机制
支持操作符重载 不支持操作符重载
支持无符号数 没有无符号数

C++主要用于系统性应用程序的设计, 大型游戏和桌面应用；
Java语法更清晰，规模更小，跨平台，适用于服务端和客户端应用程序开发。

如果在面试中对某种语言的某种实现不太了解，试着思考自己熟悉的语言是怎么实现的；如果要自己来实现会如何做等。如果实在想不出来，试着向面试官解释自己暂时不记得了，但是自己可以通过一个例子来展示它的用法，例如这段代码……

Chapter 11 Testing

测试相关的问题不仅仅只会出现在测试工程师的面试中，而且会出现在一般的程序员面试中，用来考察程序员对代码测试的能力。一个好的程序员要知道如何测试自己写的代码。
常见的测试问题包括：

  1. 解释如何测试真实世界中的物体（笔，夹子等）
  2. 解释如何测试软件（浏览器）
  3. 编写测试代码

针对第一种情况，我们可以举出一个例子：问如何测试夹子（paperclip）?
遇到这种问题，首先应该弄清楚：这个夹子的用途是什么，以及使用的人群是哪类？ 这个夹子可能是给教师拿来夹试卷的，也可能是给手工艺者拿来设计折纸的。不同的用途会有不同的答案。在确保弄清楚用途和使用者后，接着要考虑的是：

  1. 测试用例的范围是什么？例如：夹两张纸，最多夹30张纸。
  2. 如何定义不合格？
  3. 除了要求的用途外，是否还有其他用途。
  4. 测试的极端条件是什么什么？高温，低温等
