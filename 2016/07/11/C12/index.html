Chapter 13/14 C++ & JAVA

一般来说，面试官不会关心你用什么语言写代码。但是如果你在简历里面说了自己会某种语言, 则面试官可能会问你相关的问题。不过不用担心，你并不需要很清楚语言的细节，因为面试官也不一定很懂。但是你需要知道基本知识，例如c++中的指针和Java中的类等。

C++ Java
面向过程/对象 完全面向对象
编译型语言,生成的目标文件针对特定的cpu 解释型语言, 类文件平台无关
支持多重继承 单一继承
可以使用指针访问内存 无法通过指针访问内存
程序员管理堆内存 自动内存管理, 垃圾回收机制
支持操作符重载 不支持操作符重载
支持无符号数 没有无符号数

C++主要用于系统性应用程序的设计, 大型游戏和桌面应用；
Java语法更清晰，规模更小，跨平台，适用于服务端和客户端应用程序开发。

如果在面试中对某种语言的某种实现不太了解，试着思考自己熟悉的语言是怎么实现的；如果要自己来实现会如何做等。如果实在想不出来，试着向面试官解释自己暂时不记得了，但是自己可以通过一个例子来展示它的用法，例如这段代码……

13.2 Compare and contrast a hash table vs an STL map How is a hash table implemented? If the number of inputs is small, what data structure options can be used instead of a hash table?

hash table: 实值的存储位置是由其键值对应的哈希函数值来确定的，因此存储在哈希表中的数据并不有序。查找/插入时间复杂度为O(1) (假设冲突很少), 另外必须处理好冲突。
stl map: 底层存储结构是红黑树，因此数据是有序的。插入/查找时间复杂度为O(logn), 适用于查找最大/小值, 按顺序输出, 找某个特定的值, 无需处理冲突情况。

hash table的实现需要实现哈希函数(确保哈希值均匀分布)，冲突处理方法，一般有拉链法和探测法，以及当装载因子大于某个阈值时动态增加表的大小等。

如果输入数据的规模不大的话，推荐用stl map, 因为此时logn的查找和插入时间复杂度可接受，空间的利用率也高。

13.3 How do virtual functions work in C++?

通过虚函数表和一个指向它的指针，实现动态联编。

14.1 In terms of inheritance, what is the effect of keeping a constructor private?

当声明类的构造函数为私有时，由于在类的外部不能直接访问类的私有成员，所以不能通过外部构造一个对象。因此，唯一创建对象的方法是通过类的静态的公有方法，可以不必产生对象而调用它们。这样做是为了保证其他类不能从这个类派生或者创建类的实例，还可以指定类对象的个数。

14.2 In Java, does the finally block gets executed if we insert a return statement inside the try block of a try-catch-finally?

可以。函数中如果try得到了执行，那么即使在try中返回，退出，抛出异常等，finally也会得到执行。只有在try执行的过程中jvm退出或者线程被杀死, finally才不会执行。

14.3 What is the difference between final, finally, and finalize?

final:
修饰一般变量, 变量必须在声明时初始化，并且之后不能更改；
修饰对象(引用), 不能指向其他对象;
修饰方法，方法不可重载，重写;
修饰类，类不能被继承；
finally：
用于异常处理结构，表示总是执行
finalize:
是object类的一个方法，在垃圾回收器执行的时候会调用被回收对象的该方法，可以覆盖该方法以提供其他资源的回收，例如关闭文件等。但是jvm不能保证该方法一定会被调用。
